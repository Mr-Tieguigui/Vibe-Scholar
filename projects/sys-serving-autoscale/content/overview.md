# Auto-Scaling Policies for LLM Serving Clusters

## Definition

**Definition:** Develop and evaluate auto-scaling policies for LLM serving that balance latency SLOs, cost, and GPU utilization under variable request loads.

## Motivation

This project addresses a gap in Systems & Tooling research. Develop and evaluate auto-scaling policies for LLM serving that balance latency SLOs, cost, and GPU utilization under variable request loads. Current approaches lack systematic evaluation under realistic constraints.

## Research Questions

- **Q1:** How does the proposed approach compare to existing baselines?
- **Q2:** What are the key factors driving performance differences?
- **Q3:** How sensitive is the method to hyperparameter choices?
